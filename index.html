Introduccion a Git

Git es un control de versiones basado en el modelo de ramificaciones.

¿Qué es un control de versiones, y por qué debería importarte? Un control de versiones es un sistema que registra los cambios realizados en un archivo o conjunto de archivos a lo largo del tiempo, de modo que puedas recuperar versiones específicas más adelante.


Si eres diseñador gráfico o de web y quieres mantener cada versión de una imagen o diseño (es algo que sin duda vas a querer), usar un sistema de control de versiones (VCS por sus siglas en inglés) es una decisión muy acertada. Dicho sistema te permite regresar a versiones anteriores de tus archivos, regresar a una versión anterior del proyecto completo, comparar cambios a lo largo del tiempo, ver quién modificó por última vez algo que pueda estar causando problemas, ver quién introdujo un problema y cuándo, y mucho más. Usar un VCS también significa generalmente que si arruinas o pierdes archivos, será posible recuperarlos fácilmente. Adicionalmente, obtendrás todos estos beneficios a un costo muy bajo.

supongamos que tu proyecto es un arbol en pleno crecimiento, cada dia que vas avanzando el tronco va creciendo y haciendose mas grande, a medida que el tronco va creciendo le van saliendo ramas las cuales creas tu con el mismo proyecto que lleva el tronco pero en una instancia diferente, en esa instancia puedes hacer cambios sin que afectes al tronco principal, por lo que puede crear versiones diferentes de tu proyecto y si te gusta puedes unirlas a la rama principal o tronco, git te permite ir creando un historial de tu proyecto, como una linea de tiempo, puedes observar como era tu proyecto hace dias atras y lo mejor es que si el proyecto te gustaba como estaba antes puedes regresar o puedes ir al futuro, con git eso es posible.


git te permite tener un trabajo colaborativo con todos los de tu equipo, sin importar que lejos esten, en que planeta se encuentren, despues que tengas una conexion a internet pueden unirse a git y trabajar como uno solo.

Despues de todo en equipo es mejor

Si tienes un proyecto puedes subirlo a git (Repositorios es el nombre que recive tu proyecto en git) y todo el mundo puede contribuir para mejorar tu proyecto, llevandose copias de tu trabajo a sus computadoras y luego subiendolas de nuevo a tu repositorio, asi tu te llevas esos cambios a tu computatora personal y miras como va quedando.

Pueden salir obras de arte cuando todo el mundo colobara entre si

Tu identidad en git

como en toda red social, sitio web o plataforma tienes la posibilidad de registrarte con algunos datos personales y de contacto, como puede ser correo, contraseña o nombre de usurio, en git no es la exepcion, git pide configurar unos datos personales como son, usuario y correo electrónico o e-mail. estos datos son importantes ya que a la hora de estar reaizando un proyecto o estar dentro de uno, estos datos sirve para conocer que miembro del equipo iso algun cambio o "commit".

con el comando git config --list podras ver el estado de tu perfil en git, de seguro estara sin nombre y correo porque no lo hemos configurado


Vamos a configurar nuestros datos en git, para eso utilizaremos dos simples comandos, uno para el nombre de usuario y otro para nuestro e-mail o correo


git config --global user.name "juan david"
git config --global user.email johndoe@example.com

en en primer codigo, lo que esta denro de las comillas sera el nombre que elijamos como nuestro usuario, y luego el segundo comando es para introduccir nuestro e-mail

con esto ya estaran las primeras configuraciones y lo mejor es que solo se hace este proceso una sola vez, ya que con el comando --global no tendras que repetir el proceso y se utilizaran estos datos en todo lo que hagas, a menos de que cambies de ordenador e instales git de nuevo.

si quieremos cambiar estos datos debemos usar los mismos comandos pero sin el --global
