Git / github

es un software de control de versiones diseñado por Linus Torvalds, pensando en la eficiencia, la confiabilidad y compatibilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran número de archivos de código fuente. Su propósito es llevar registro de los cambios en archivos de computadora incluyendo coordinar el trabajo que varias personas realizan sobre archivos compartidos en un repositorio de código.

Introduccion a Git

Git es un control de versiones basado en el modelo de ramificaciones.

¿Qué es un control de versiones, y por qué debería importarte? Un control de versiones es un sistema que registra los cambios realizados en un archivo o conjunto de archivos a lo largo del tiempo, de modo que puedas recuperar versiones específicas más adelante.


Si eres diseñador gráfico o de web y quieres mantener cada versión de una imagen o diseño (es algo que sin duda vas a querer), usar un sistema de control de versiones (VCS por sus siglas en inglés) es una decisión muy acertada. Dicho sistema te permite regresar a versiones anteriores de tus archivos, regresar a una versión anterior del proyecto completo, comparar cambios a lo largo del tiempo, ver quién modificó por última vez algo que pueda estar causando problemas, ver quién introdujo un problema y cuándo, y mucho más. Usar un VCS también significa generalmente que si arruinas o pierdes archivos, será posible recuperarlos fácilmente. Adicionalmente, obtendrás todos estos beneficios a un costo muy bajo.

supongamos que tu proyecto es un arbol en pleno crecimiento, cada dia que vas avanzando el tronco va creciendo y haciendose mas grande, a medida que el tronco va creciendo le van saliendo ramas las cuales creas tu con el mismo proyecto que lleva el tronco pero en una instancia diferente, en esa instancia puedes hacer cambios sin que afectes al tronco principal, por lo que puede crear versiones diferentes de tu proyecto y si te gusta puedes unirlas a la rama principal o tronco, git te permite ir creando un historial de tu proyecto, como una linea de tiempo, puedes observar como era tu proyecto hace dias atras y lo mejor es que si el proyecto te gustaba como estaba antes puedes regresar o puedes ir al futuro, con git eso es posible.


git te permite tener un trabajo colaborativo con todos los de tu equipo, sin importar que lejos esten, en que planeta se encuentren, despues que tengas una conexion a internet pueden unirse a git y trabajar como uno solo.

Tu identidad en git

como en toda red social, sitio web o plataforma tienes la posibilidad de registrarte con algunos datos personales y de contacto, como puede ser correo, contraseña o nombre de usurio, en git no es la exepcion, git pide configurar unos datos personales como son, usuario y correo electrónico o e-mail. estos datos son importantes ya que a la hora de estar reaizando un proyecto o estar dentro de uno, estos datos sirve para conocer que miembro del equipo iso algun cambio o "commit".

con el comando git config --list podras ver el estado de tu perfil en git, de seguro estara sin nombre y correo porque no lo hemos configurado


Vamos a configurar nuestros datos en git, para eso utilizaremos dos simples comandos, uno para el nombre de usuario y otro para nuestro e-mail o correo


git config --global user.name "juan david"
git config --global user.email johndoe@example.com

en en primer codigo, lo que esta denro de las comillas sera el nombre que elijamos como nuestro usuario, y luego el segundo comando es para introduccir nuestro e-mail

con esto ya estaran las primeras configuraciones y lo mejor es que solo se hace este proceso una sola vez, ya que con el comando --global no tendras que repetir el proceso y se utilizaran estos datos en todo lo que hagas, a menos de que cambies de ordenador e instales git de nuevo.

si quieremos cambiar estos datos debemos usar los mismos comandos pero sin el --global

Iniciando un proyecto con git

ya tenemos todo listo para poder trabar con el sistema de control de versiones, ahora solo toca crear un proyecto e importarlo en git, esto se hace de manera muy simple por medio de la consola con un comando muy simple.

Como ya sabemos los proyectos que se crean o se suben a git se les llama repositorios, los cuales por su nombre lo indica iran reposando todo los archivos, cambios e informacion de un proyecto. existen 2 tipos de repositorios, estan los remotos y los locales.

Los repositorios remotos son versiones de tu proyecto que están hospedadas en Internet o en cualquier otra red. Puedes tener varios de ellos, y en cada uno tendrás generalmente permisos de solo lectura o de lectura y escritura.

Los repositorios locales son versiones de tu proyecto que estan hospedados en tu ordenador o computadora personal, es decir se gurdan en nuestra pc y podemos tener accesos a el en cualquier momento y sin conexion a internet

el nombre puee ser cualquiera, ahora que ta tenemos lista la carpeta, vamos a proceder a crear los archivos del proyecto, para eso voy a recrear un ejemplo creando dos archivos que serviran para crear una pagina web, utilizando mi editor de codigo preferido que sera visual estudio code

Para inicializar o comenar el repositorio debemos ubicarnos dentro de la carpeta de nuestro proyecto, en este caso lo aremos por consola de comandos pero existen formas mucho mas sencillas de iniciar un repo sin tener que estar metidos tanto con comandos que se pueen olvidar, como es el caso de vs code que te permite trabajar con git de una manera mucho mas facil, mas adelante mostrare como se hace.

Abrimos nuestra consola click izquiero y en git bash here



usaremos la consola git bash que es la apropiada utilizar cuando se trabaja con git





se abrira la consola con el nombre de la carpeta en donde ira el proyecyo. ahora utilizaremos un comando muy simple

Se inicializara el proyecto en local y ya hemos iniciado a trabajar con git

Ahora utilizaremos otro comando para ver en que estado esta nuestro proyecto

git status



se mostraran los dos archivos que hemos creado en color rojo, ¿que significa esto?

Los archivos en rojo significan que git no esta haciendo un seguimiento y que por lo tanto cualquier cambio que se haga en dichos archivos no se guardara en git, por lo tanto debemos utilizar otro comando basico para que git les haga seguimiento.

git add .

con ese comando se le esta diciendo a git que haga un seguimiento de todos los archivos, si quisieramos seguir a un solo archivo utilizaremos el mismo comando pero esta ves en ves del punto espesificamos que archivo seguir, ejemplo.

git add index.html

Lo mas comun es usar el primer comando, para asi no tener que estar espesificando todo el tiempo que archivo seguir. despues de haber tecleado el comando git ya estara siguiendo todos los archivos del proyecto, puedes comprobarlo escribiendo de nuevo el git status



Si nos damos cuenta los archivos estan en verde por lo que el seguimiento de git esta activado, ahora ya podemos empezar a desarrollar nuestro proyecto.

Simpre que agregemos un nuevo archivo utilizaremos el git add . para controlar mantener el control de todos los archivos que vamos a seguir con git.

Pero con ese comando solo estamos haciendole un seguimiento a los archivos, mas no gurdando los cambios que hagamos, es por eso que aqui entra otro comando basico que te permitira guardar todos los cambios realizados y colocarles un comentario para tener un registro ordenado de cambios. El famoso commit en git es el encargado de registrar los cambios de un proyecto, por eso cuando se modifique alguna cosa en los archivos es necesario hacer un commit para que los cambios de guarden en el repositorio. 

git commit -m "comentarios del commit"

dentro de las comillas iran comentarios personalizados que queramos agregar para ir registrando como si fuera un itinerario.

Para ver el historial de cambios o commit coloamos el comando 

git log

Repositorios Remotos

Para poder colaborar en cualquier proyecto Git, necesitas saber cómo gestionar repositorios remotos. Los repositorios remotos son versiones de tu proyecto que están hospedadas en Internet o en cualquier otra red. Puedes tener varios de ellos, y en cada uno tendrás generalmente permisos de solo lectura o de lectura y escritura. Colaborar con otras personas implica gestionar estos repositorios remotos enviando y trayendo datos de ellos cada vez que necesites compartir tu trabajo. Gestionar repositorios remotos incluye saber cómo añadir un repositorio remoto, eliminar los remotos que ya no son válidos, gestionar varias ramas remotas, definir si deben rastrearse o no y más. En esta sección, trataremos algunas de estas habilidades de gestión de remotos.

Son muchos comandos que se utilizan cuando se crean o usan repositorios remotos, es por eso que tratare de resumir lo mas basico y mas usado de git.
